## 二、大模型选择：免费API组合策略

### 2.1 各平台免费额度对比（2026年5月最新）

| 平台 | 免费额度 | 有效期 | 代表模型 | 上下文 | 并发/RPM | 推荐度 |
|------|---------|--------|---------|--------|---------|--------|
| **智谱AI** | GLM-4-Flash永久免费 + 新用户2000万token | 永久 | GLM-4-Flash, GLM-4.7-Flash | 128K-200K | 30并发 | ⭐⭐⭐⭐⭐ |
| **硅基流动** | 新用户2000万token + 9B以下模型永久免费 | 永久 | Qwen3-8B, DeepSeek-R1, GLM-4-9B | 各模型不同 | 1000 RPM | ⭐⭐⭐⭐⭐ |
| **火山方舟** | 每模型每天50万token | 按天重置 | 豆包全系, DeepSeek-V3, GLM-4 | 256K | 1万RPM(lite) | ⭐⭐⭐⭐ |
| **百度千帆** | Lite/Speed-8K永久免费 | 永久 | ERNIE-Speed-8K, Speed-128K | 8K-128K | 300 RPM | ⭐⭐⭐⭐ |
| **腾讯混元** | Hunyuan-lite永久免费 + 100万token | 1年 | Hunyuan-lite, Hunyuan-TurboS | 各不同 | 较低 | ⭐⭐⭐ |
| **阿里百炼** | 新用户7000万tokens | 90天 | Qwen-plus全系 | 各不同 | 中等 | ⭐⭐⭐ |
| **DeepSeek官方** | 新用户100万token | 30天 | DeepSeek-V4, R1 | 各不同 | 较低 | ⭐⭐⭐ |
| **Gemini** | 2.5 Flash 每天1440次请求 | 永久 | Gemini 2.5 Flash | 100万+ | 15 RPM(lite) | ⭐⭐⭐ |

### 2.2 关键发现

**火山方舟的隐藏福利**：VolcProxy项目揭示了火山方舟每模型每天50万token的免费额度，配置7+个模型即可获得350万+ tokens/天。加上Coding Plan的300次/5小时，总量可观。

**硅基流动的价值**：9B以下模型永久免费意味着Qwen3-8B、GLM-4-9B等可以无限调用，且1000 RPM并发极高，适合高频Agent调用。

**智谱AI是定海神针**：GLM-4-Flash永久免费，128K上下文，30并发，是最可靠的兜底方案。

### 2.3 推荐降级链策略

```
请求 → ① 硅基流动(Qwen3-8B, 永久免费, 高并发)
      → ② 智谱AI(GLM-4-Flash, 永久免费, 128K)
      → ③ 火山方舟(豆包-lite/DeepSeek-V3, 日重置50万token)
      → ④ 百度千帆(Speed-8K, 永久免费)
      → ⑤ 腾讯混元(lite, 永久免费)
      → ⑥ Coze v3 API(备用, 消耗积分)
```

**降级触发条件**：
- 返回429(Rate Limit) → 切换下一个
- 返回402/403(额度耗尽) → 跳过该平台
- 超时5秒 → 重试1次后切换
- 网络错误 → 切换下一个

### 2.4 每交互Token消耗估算

| 交互类型 | 平均Token消耗 | 每日预估次数 | 日消耗 |
|---------|-------------|------------|--------|
| 简单对话 | ~2000 | 50 | 10万 |
| Agent工具调用 | ~5000 | 20 | 10万 |
| 复杂推理 | ~10000 | 5 | 5万 |
| **日合计** | | | **~25万token** |

**结论**：日消耗25万token，仅硅基流动永久免费模型(1000 RPM)即可覆盖。加上智谱AI和火山方舟，免费额度充裕到几乎不可能用完。

### 2.5 本地跑小模型：不可行

| 设备 | RAM | 最低要求 | 结论 |
|------|-----|---------|------|
| iPhone X | 3GB | 需4GB+ | ❌ 无法运行任何LLM |
| iPhone 8 | 2GB | 需4GB+ | ❌ 完全不可能 |

- Fullmoon/PocketPal AI 均需A12+芯片和Metal 3支持
- A11芯片不支持MLX框架
- 即使勉强加载0.6B模型，质量极差，无实用价值
- **结论**：本地模型不可行，依赖免费API是唯一出路

## 三、记忆系统

### 3.1 当前状态

- 记忆文件：SOUL.md, USER.md, MEMORY.md, TOOLS.md
- 本地读写已验证（App通过Tweak shell命令读取）
- 记忆文件位于越狱文件系统，App可直接访问

### 3.2 推荐方案：增强文件系统 + 结构化索引

**不需要向量数据库/RAG**，理由：
1. 记忆文件总量小（几十KB级别），全文检索即可
2. 向量数据库在iPhone上增加复杂度但收益有限
3. 免费API的128K上下文窗口已足够承载所有记忆

**推荐方案**：

```
记忆文件结构:
/var/mobile/StarCore/
├── SOUL.md              # 人格定义 (不变)
├── USER.md              # 用户画像 (不变)  
├── MEMORY.md            # 核心记忆 (结构化增强)
├── TOOLS.md             # 工具注册表 (不变)
├── memory_index.json    # 新增: 结构化索引
└── conversations/       # 新增: 对话历史
    ├── 2026-05-15.jsonl
    ├── 2026-05-14.jsonl
    └── ...
```

**memory_index.json 结构**：

```json
{
  "last_updated": "2026-05-15T10:30:00+08:00",
  "topics": [
    {"topic": "股票分析", "last_mentioned": "2026-05-15", "count": 5},

```

### 3.3 记忆注入策略

每次LLM调用时，将记忆文件内容作为system prompt的一部分注入：


## 四、Agent调度

### 4.1 Hermes-Lite vs App内置调度

| 维度 | Hermes-Lite | App内置调度 |
|------|-------------|------------|
| 运行稳定性 | Python进程可能被iOS杀死 | Swift原生，与App同生命周期 |
| 资源占用 | 额外占用50-100MB RAM | 几乎无额外开销 |
| 启动速度 | 需要python3启动 | App启动即就绪 |
| 定时任务 | 内置cron，但需进程存活 | iOS Background Tasks受限 |
| Dashboard | ✅ 8642端口Web管理 | 需自行实现 |
| 多模型支持 | ✅ 已有配置 | 需自行实现降级链 |
| 网关能力 | Gateway未启动，配置复杂 | 不需要 |

**推荐方案**：**App内置调度为主，Hermes-Lite为辅助**

理由：
1. Hermes-Lite的Gateway始终未启动，核心价值打了折扣
2. App内置调度更稳定，不受iOS杀进程影响
3. Hermes-Lite Dashboard可用于调试和监控（8642端口保留）
4. 定时任务依赖launchd而非Python进程

### 4.2 App内置调度设计

```swift
// 简化的状态机调度
enum AgentState {
    case idle           // 等待指令
    case thinking       // LLM推理中
    case acting         // 执行工具调用
    case waiting        // 等待用户确认
    case offline        // 无网络降级模式
}

class AgentScheduler {
    let llmChain = LLMFallbackChain()  // 多模型降级链
    let memory = MemoryManager()        // 记忆管理
    let toolExecutor = ToolExecutor()   // 工具执行
    
    // 核心循环
    func processInput(_ input: String) async -> String {
        let context = memory.buildContext()
        let response = await llmChain.chat(input, context: context)
        if let toolCalls = response.toolCalls {
            let results = await toolExecutor.execute(toolCalls)
            return await processToolResults(results)
        }
        memory.saveInteraction(input, response)
        return response.text
    }
}
```

### 4.3 定时任务方案

**推荐：iOS launchd (越狱环境)**

越狱环境下可创建launchd plist实现真正的后台定时任务：

```xml
<!-- /Library/LaunchDaemons/com.starcore.scheduled.plist -->
<dict>
    <key>Label</key>
    <string>com.starcore.scheduled</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/bin/python3</string>
        <string>/var/mobile/StarCore/scheduled_tasks.py</string>
    </array>
    <key>StartInterval</key>
    <integer>3600</integer>  <!-- 每小时 -->
    <key>RunAtLoad</key>
    <true/>
</dict>
```

**定时任务示例**：
- 每小时：检查新消息 → 调用LLM生成回复
- 每6小时：压缩记忆文件，生成摘要
- 每天0点：重置火山方舟token计数，清理旧对话
- 每周一：分析对话历史，更新用户画像

**Hermes scheduled jobs 作为备选**：如果Hermes-Lite Gateway能启动成功，可用其cron功能。

## 五、通信架构

### 5.1 通信矩阵

| 通道 | 协议 | 端口 | 状态 | 用途 |
|------|------|------|------|------|
| App ↔ Tweak | TCP | 6000 | ✅已通 | 触摸注入、键盘模拟 |
| App ↔ ios-mcp | HTTP | 8090 | ✅已通 | 截图、UI树、App管理 |
| App ↔ LLM | HTTPS | 443(出站) | ✅已通 | 大模型对话 |
| App ↔ Coze | HTTPS | 443(出站) | ✅备用 | 云端超脑(降级末位) |
| iPhone X ↔ iPhone 8 | HTTP | 自定义 | ⚠️需搭建 | 跨设备协调 |
| App ↔ Hermes Dashboard | HTTP | 8642 | ✅可用 | 调试监控 |
| App ↔ ESP32 | HTTP/BLE | — | ❌未通 | IoT控制 |

### 5.2 Tweak是否需要监听0.0.0.0？

**推荐：保持127.0.0.1，但为iPhone 8场景添加可选远程模式**

理由：
- 安全：0.0.0.0暴露给热点网络，其他设备可注入触摸事件
- Tweak和App在同一设备，127.0.0.1已够用
- iPhone 8需要操作iPhone X时，走ios-mcp的8090端口（已支持远程）

**如果确实需要iPhone 8控制iPhone X的Tweak**：
1. 在Tweak配置中添加`listen_address`选项
2. 默认127.0.0.1，手动切换到0.0.0.0
3. 添加简单Token认证机制

### 5.3 两台iPhone之间通信

```
iPhone X (172.20.10.1) ←── WiFi热点 ──→ iPhone 8 (172.20.10.2)
         │                                      │
         │  HTTP API (推荐)                      │
         │  GET /api/execute                     │
         │  POST /api/touch                      │
         │  GET /api/screenshot                   │
         │                                       │
         │  iPhone 8作为"执行臂":                  │
         │  - TouchSprite PE自动化                │
         │  - ios-mcp远程控制                     │
         │  - 辅助计算/下载                        │
```

**实现**：iPhone 8上运行简单的Flask/FastAPI服务：

```python
# iPhone 8: /var/mobile/StarCore/agent_node.py
from fastapi import FastAPI
import subprocess

app = FastAPI()

@app.get("/health")
def health():
    return {"status": "ok", "device": "iPhone8"}

@app.post("/touch")
def touch(x: int, y: int, action: str = "tap"):
    # 通过ios-mcp或TouchSprite执行
    result = subprocess.run(["curl", "-s", f"http://127.0.0.1:8090/tap_screen?x={x}&y={y}"], capture_output=True)
    return {"result": result.stdout}

@app.get("/screenshot")
def screenshot():
    result = subprocess.run(["curl", "-s", "http://127.0.0.1:8090/screenshot"], capture_output=True)
    return {"image": result.stdout}
```

- 仅在所有免费API都不可用时使用
- 用于接收"我"的指令（反向通道）
- 通过Coze Bot的对话接口实现双向通信

## 六、完全离线场景

### 6.1 离线能力矩阵

| 功能 | 有网络 | 无网络 | 降级方案 |
|------|--------|--------|---------|
| LLM对话 | ✅ 免费API全量 | ❌ 无模型可用 | 预缓存回复模板 |
| 触摸注入 | ✅ Tweak TCP | ✅ 本地TCP | 无需降级 |
| 截图/UI树 | ✅ ios-mcp | ✅ 本地HTTP | 无需降级 |
| 记忆读写 | ✅ 文件系统 | ✅ 文件系统 | 无需降级 |
| App管理 | ✅ ios-mcp | ✅ 本地HTTP | 无需降级 |
| Shell命令 | ✅ Tweak执行 | ✅ 本地执行 | 无需降级 |
| 跨设备协调 | ✅ WiFi热点 | ⚠️ 需热点 | 蓝牙备选 |
| 云端超脑 | ✅ Coze API | ❌ 不可用 | 离线Agent模式 |

### 6.2 离线Agent模式

无网络时，App进入"离线Agent模式"：

1. **规则引擎**：基于预设规则执行简单任务
   - 定时截图 + OCR识别 + 本地通知
   - 基于UI元素树的简单判断和操作
   - 预定义的App操作序列（如打开特定App、点击特定位置）

2. **预缓存模板**：
   - 常用操作的命令模板
   - 紧急情况的SOP
   - 最近一次对话的关键信息摘要

3. **网络恢复自动升级**：
   - Reachability监听网络状态
   - 网络恢复后自动切换到在线模式
   - 离线期间的待处理任务队列自动执行

### 6.3 最佳体验（有网络时）

- 完整的Agent能力：感知→推理→行动→学习
- 多模型降级保证99.9%可用性
- 两设备协同：iPhone X思考+iPhone 8执行
- 记忆持续积累，跨会话学习

## 七、ESP32-S3的角色

### 7.1 当前局限

- 小智固件2.0.1，所有本地端口关闭
- 只出站连xiaozhi.me
- 与iPhone不在同一网段（192.168.1.x vs 172.20.10.x）
- 不支持BLE HID

### 7.2 方案评估

| 方案 | 可行性 | 工作量 | 价值 | 推荐 |
|------|--------|--------|------|------|
| 维持小智固件 | ✅ 已可用 | 零 | 低(仅语音入口) | 短期 |
| 刷自定义固件 → WiFi串口 | ✅ 可行 | 中 | 中 | 中期 |
| 刷自定义固件 → BLE键盘 | ⚠️ iOS限制多 | 高 | 中 | 不推荐 |
| WiFi桥接(ESP作为中间人) | ✅ 可行 | 低 | 高 | ⭐推荐 |
| 放弃ESP32，专注iPhone | ✅ | 零 | — | 备选 |

### 7.3 推荐方案：ESP32 WiFi桥接

**核心思路**：刷自定义固件，让ESP32-S3同时连接家庭WiFi和iPhone热点，充当网络桥接器。

```
ESP32-S3 自定义固件:
  STA模式1: 连接家庭WiFi (192.168.1.x) → 访问xiaozhi.me
  STA模式2: 连接iPhone X热点 (172.20.10.x) → 被iPhone访问
  
  或者更简单:
  STA模式: 连接iPhone X热点
  HTTP Server: 172.20.10.x:8080 → 接收指令
  功能: 语音输入/输出, 传感器数据, GPIO控制
```

**但是**：ESP32-S3只支持一个STA连接，需要AP+STA模式：
- STA: 连家庭WiFi（保持小智语音功能）
- AP: 开放热点给iPhone连接
- 问题：iPhone需要同时连两个WiFi → 不现实

**最终推荐**：

1. **短期**：维持小智固件不变，ESP32仅作为语音助手入口
2. **中期**：刷ESP-IDF自定义固件，改为连iPhone热点，开放HTTP端口
   - 语音唤醒 → 录音 → 发送到iPhone → LLM处理 → TTS播放
   - 传感器数据上报
   - GPIO控制（未来扩展智能家居）
3. **不推荐BLE HID方案**：iOS对BLE HID限制严格，需开启辅助触控，体验差

### 7.4 投入建议

**ESP32-S3不值得大量投入**，理由：
- iPhone X本身的ios-mcp + Tweak已覆盖所有触摸/UI操作
- BLE HID在iOS上体验差，不如直接Tweak注入
- 语音入口可暂时通过小智固件实现
- 精力应集中在App架构优化和免费API集成上

## 十、风险点与备选方案

### 风险1：免费API额度突然收紧

**概率**：中等（历史上发生过）
**影响**：高
**备选**：
- 注册多个账号轮换（智谱AI允许手机号注册）
- 阿里云百炼7000万tokens作为储备
- DeepSeek官方API价格极低（0.27元/百万token），作为付费兜底
- VolcProxy式自动轮换方案

### 风险2：iOS越狱补丁失效

**概率**：低（Dopamine稳定，iOS 16.7.12不再更新）
**影响**：致命
**备选**：
- 使用ios-mcp替代Tweak进行操作（不需要注入SpringBoard）
- TouchSprite PE不依赖越狱
- 降级为"纯App模式"，仅保留LLM对话能力

### 风险3：iPhone X性能不足

**概率**：中等
**影响**：中等
**备选**：
- 减少App内存占用：限制对话上下文长度
- 将部分计算卸载到iPhone 8
- 关闭不必要的后台服务

### 风险4：4G网络不稳定

**概率**：中等
**影响**：高
**备选**：
- 离线Agent模式（规则引擎）
- 预缓存关键信息
- 连接家庭WiFi时预加载

### 风险5：Hermes-Lite Dashboard进程被杀

**概率**：高
**影响**：低（仅影响调试）
**备选**：
- launchd保活配置
- App内置简单Dashboard替代
- 关键监控逻辑移入App

## 十一、关键决策总结

| 决策点 | 选择 | 理由 |
|--------|------|------|
| LLM方案 | 免费API组合 | 本地跑模型不可行(3GB RAM)，免费API充裕 |
| 记忆方案 | 增强文件系统 | 简单可靠，无需向量数据库 |
| 调度方案 | App内置为主 | 比Hermes-Lite更稳定，生命周期一致 |
| 定时方案 | launchd | 越狱环境独有优势，可靠后台执行 |
| ESP32方案 | 维持现状 | 投入产出比低，iPhone自身能力已够 |
| 编译方案 | GitHub Actions | 零成本，替代云电脑 |
| 通信方案 | 本地TCP/HTTP | 不再需要Cloudflare隧道 |
| Tweak监听 | 保持127.0.0.1 | 安全优先，跨设备走ios-mcp |
