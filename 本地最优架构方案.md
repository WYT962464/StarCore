# StarCore 最优架构方案

> 重写日期：2026-05-16
> 核心目标：**本地AI操控手机**——说话→AI想→动手操作→直到完事

## 一、是什么

```
星核 = AI操控手机

你说话 → LLM想 → 输出action → Tweak执行 → 结果喂回LLM → 继续想 → 完事
```

就4个东西：

| 模块 | 是什么 | 接口 | 状态 |
|------|--------|------|------|
| **界面** | App聊天界面，说话看结果 | 用户交互 | ✅已有 |
| **脑子** | 火山方舟API | HTTPS出站 | ✅已有 |
| **手脚** | Tweak触摸注入+shell | TCP 127.0.0.1:6000 | ✅已部署 |
| **眼睛** | ios-mcp截图+UI树 | HTTP 127.0.0.1:8090 | ✅已部署 |
| **记忆** | md文件系统 | /var/mobile/StarCore/ | ✅已有 |

## 四、手脚：Tweak + ios-mcp

### Tweak (TCP :6000) — 主力，速度快

| action | 参数 | 说明 |
|--------|------|------|
| tap | x, y (归一化0-1) | 点击 |
| swipe | fromX,fromY,toX,toY,duration | 滑动 |
| shell | command | 执行shell命令 |
| openApp | bundleId | 打开App |
| pressHome | - | Home键 |
| pressPower | - | 电源键 |
| pressVolumeUp/Down | - | 音量键 |
| screenshot | - | 截图(最快) |
| inputText | text | 中文输入(剪贴板粘贴) |
| typeText | text | 英文逐字输入 |
| getScreenInfo | - | 屏幕尺寸+当前App |

### ios-mcp (HTTP :8090) — 备选，功能更全

截图、UI元素树、App列表、安装/启动App等。Tweak搞不定时用这个。

## 六、核心循环代码逻辑

```swift
// 伪代码
func agentLoop(userInput: String) {
    messages = [systemPrompt + 记忆文件] + 对话历史 + [userInput]
    
    while (任务没完成 && step < 20) {
        reply = callLLM(messages)  // 流式输出
        
        actions = 从reply中提取JSON action
        
        if (没有action) { break }  // AI说完了
        
        for action in actions {
            result = 执行action(tweak/mcp)
            用户看到: "→ 步骤N: 操作结果"
        }
        
        // 结果喂回LLM，继续想
        messages.append(AI的回复)
        messages.append(操作结果)
    }
}
```

## 七、砍掉清单

| 砍掉 | 原因 |
|------|------|
| 云桥/超脑/Coze API | 本地能做不上云 |
| 6平台降级链 | 火山方舟2000万tokens/天一家够用 |
| 访客模式 | 从来没成功过 |
| Hermes-Lite | App是完整方案，Hermes没手脚是多余的 |
| Cloudflare隧道 | 全本地通信，不需要公网暴露 |
| ESP32 | 投入产出比低 |
| iPhone 8辅助节点 | 现阶段不需要 |
| MCP协议协商 | 直接HTTP调ios-mcp就行 |

## 八、实施路线

### 第一步：跑通核心循环

1. App聊天界面能收到LLM回复 ✅ (v10.0已切DeepSeek-V3.2)
2. LLM输出JSON action → Tweak执行 → 结果回传 → 循环继续
3. 清理云控/超脑/Hermes代码，App减负
4. 验证：说"帮我截个图"→ AI调screenshot → 图出来了

### 第二步：稳定+增强

1. 火山方舟内部降级链（endpoint切换）
2. 记忆注入system prompt
3. 截图→发给LLM→LLM看屏幕决定下一步
4. 每步操作间隔自适应（等App加载完再操作）
