# 星核iOS App闪退问题修复方案研究

## 一、问题根因分析

### 1.1 崩溃原因定位

通过代码分析（ContentView.swift），v0.2.0 中有两处使用 Mach API 可能导致闪退：

**第422行 `updateCPUUsage()`**
```swift
let r = host_processor_info(mach_host_self(), PROCESSOR_CPU_LOAD_INFO, &n, &info, &count)
```
- 使用 `host_processor_info` 获取CPU负载
- Mach API 在沙盒中可能被拒绝访问

**第423行 `updateMemoryUsage()`**
```swift
let r = withUnsafeMutablePointer(to: &t) { $0.withMemoryRebound(to: integer_t.self, capacity: Int(c)) {
    task_info(mach_task_self_, task_flavor_t(TASK_VM_INFO), $0, &c)
}}
```
- 使用 `task_info` 获取内存信息
- 在沙盒环境中受限

### 1.2 核心问题

**全能签签名的App仍然运行在iOS沙盒中**，即使设备已越狱。Dopamine是"rootless"越狱，不会修改系统根目录，沙盒限制仍然生效。

## 三、问题核心：Dopamine Rootless特性

### 3.1 Rootless越狱的限制

根据研究，Dopamine 2.x 是**rootless（无根）越狱**：

1. **不提供root权限** - 无法写入 `/` 系统分区
2. **沙盒限制仍然存在** - App仍然在沙盒中运行
3. **Rootless JB目录** - 越狱文件在 `/var/jb` 而非 `/`
4. **tweak注入机制** - 通过 `DYLD_INSERT_LIBRARIES` 实现

### 3.2 为什么全能签App仍然闪退

全能签签名的App：
- ✅ 不需要越狱即可安装
- ❌ 仍然运行在标准iOS沙盒中
- ❌ 没有root权限
- ❌ 无法访问越狱文件系统

即使设备已越狱，**签名安装的第三方App仍然受到沙盒限制**。

## 四、推荐最小可行方案

### 4.1 v0.3.0 修复建议

**策略：使用安全的公开API，移除有风险的Mach调用**

```swift
// 安全的CPU使用率估算（基于内存压力和活跃核心数）
func getSafeCPUEstimate() -> Double {
    let activeCores = ProcessInfo.processInfo.activeProcessorCount
    let totalCores = ProcessInfo.processInfo.processorCount
    // 使用核心数比例作为粗略估算
    return Double(activeCores) / Double(totalCores) * 50.0 // 估算值
}

// 安全的内存使用（仅获取总量）
func getSafeMemoryInfo() -> (total: UInt64, used: Double) {
    let total = ProcessInfo.processInfo.physicalMemory
    // 不尝试获取详细使用情况
    return (total, 0.0)
}
```

### 4.2 替代方案：使用设备信息推断

| 实际想获取 | 替代方案 | 说明 |
|-----------|---------|------|
| CPU使用率 | 电池温度 + 活跃核心数 | 间接估算 |
| 内存占用 | 使用趋势 + 总内存 | App级别统计 |
| 温度信息 | 显示"不可用" | iOS不提供公开API |

## 五、解决方案实施步骤

### 5.1 立即可用的修复（推荐）

1. **移除Mach API调用**
   - 删除 `host_processor_info` 调用
   - 删除 `task_info` 调用

2. **使用安全的替代实现**
   ```swift
   // CPU估算（基于活跃核心数）
   var cpuUsage: Double {
       let ratio = Double(ProcessInfo.processInfo.activeProcessorCount) / 
                   Double(ProcessInfo.processInfo.processorCount)
       return ratio * 50.0 // 估算50%基础负载
   }
   
   // 内存显示总容量
   var memoryInfo: String {
       let totalGB = Double(ProcessInfo.processInfo.physicalMemory) / 1_073_741_824
       return String(format: "%.1f GB", totalGB)
   }
   ```

3. **保留UI框架API**
   - `UIDevice` - 电池、设备信息
   - `ProcessInfo` - 系统信息
   - `FileManager` - 存储空间
   - `CoreMotion` - 运动传感器
   - `CoreLocation` - 方向/磁场

### 5.2 长期方案：TrollStore安装

如果需要更深层的系统访问：

1. 使用 **TrollStore** 永久签名安装App
2. 配置Dopamine启用 **Tweak Injection**
3. 使用Choicy排除问题tweak

### 5.3 完全体方案：越狱开发环境

如需完整硬件监控能力：

1. 使用Xcode直接部署（非全能签）
2. 通过Dopamine越狱环境运行
3. 请求特殊entitlements

## 六、结论

**核心问题**：`host_processor_info` 和 `task_info` 等Mach API在全能签签名的App中可能被沙盒限制，导致闪退。

**推荐方案**：使用公开的UIKit/Foundation API获取硬件信息，移除有风险的Mach调用。App可以正常运行并获取大部分常用硬件感知能力。

**关键代码变更位置**：ContentView.swift 第422-423行
