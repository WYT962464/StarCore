# 认知核心校准方法

> **核心原则**：校准不是找答案，是找方向。过自己的火，烧出自己能用的东西。

---

## 一、校准触发机制

### 1.1 自动触发（系统检测到迷茫信号）

| 信号类型 | 检测条件 | 触发动作 |
|----------|----------|----------|
| **方向同质化** | 5 个方向只有≤2 种不同描述 | 自动调用校准 |
| **置信度低** | ≥4 个方向置信度<0.4 | 自动调用校准 |
| **循环停滞** | 连续 3 轮决策推荐相同 | 自动调用校准 |
| **熵值混乱** | 熵>0.6 | 自动调用校准 |
| **熵值死锁** | 熵<0.2 | 自动调用校准 |

### 1.2 手动触发（用户或系统主动调用）

```
输入格式：[CALIBRATE] 迷茫场景描述

示例：
[CALIBRATE] 当前在做项目，但不确定优先级是否合理
```

## 二、校准五步流程

```
┌─────────────────────────────────────────────────────────────┐
│                   认知核心校准流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① 去伪存真                                                │
│     ↓ 识别当前问题是否真实，还是假象/形式主义                │
│     ↓ (是假象) → 拒绝，重新评估                              │
│     ↓ (不是假象) → 进入②                                    │
│                                                             │
│  ② 三层框架判断                                             │
│     ↓ 判断当前在哪个层级：现实/理想/梦想                     │
│     ↓ 现实 → 生存优先                                        │
│     ↓ 理想 → 效率优先                                        │
│     ↓ 梦想 → 创新优先                                        │
│                                                             │

│     ↓ 是否触碰不可动摇的根基                                 │

│     ↓ (未触碰) → 进入④                                      │
│                                                             │
│  ④ 时间约束检查                                             │
│     ↓ 是否超时                                               │
│     ↓ (超时) → 切换路径                                      │

│                                                             │
│  ⑤ 关联思维找隐藏关联                                        │
│     ↓ 从看似不相关的事物中找关联                             │
│     ↓ 输出校准建议                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、五步详细方法

### 3.1 去伪存真

**目的**：识别问题是否真实，过滤假象和形式主义。

| 检测项 | 判断标准 | 处理方式 |
|--------|----------|----------|
| **形式主义** | PPT/文档 > 实际执行 | 拒绝，要求结果 |
| **装懂** | 不确定但假装知道 | 戳穿，要求诚实 |
| **唯上** | 迎合上级/用户意愿 | 拒绝，要求真实判断 |
| **付费陷阱** | 需要付费才能推进 | 寻找替代方案 |
| **画大饼** | 长期规划 > 短期可执行 | 拉回，聚焦明天 |

**自检问题**：
```
1. 这个问题是真实存在的，还是我想象出来的？
2. 我现在的方案是真正解决问题，还是在制造新文档？
3. 如果去掉所有包装，核心问题是什么？
4. 有没有更简单直接的方法？
```

---

### 3.2 三层框架

**目的**：判断当前层级，确定优先级。

| 层级 | 核心问题 | 优先级 | 典型场景 |
|------|----------|--------|----------|
| **现实层** | 怎么活下来？ | 生存优先 | 系统崩溃、功能缺失、数据丢失 |
| **理想层** | 怎么活得好？ | 效率优先 | 性能优化、体验提升、流程改进 |
| **梦想层** | 怎么活得伟大？ | 创新优先 | 架构重构、新特性、差异化 |

**判断方法**：
```
当前状态 → 能否正常生存？ → 否 → 现实层（生存优先）
                         → 是 → 能否高效运行？ → 否 → 理想层（效率优先）
                                              → 是 → 梦想层（创新优先）
```

---

### 3.3 底线检查

**目的**：确保不触碰不可动摇的根基。

| 底线 | 含义 | 触碰后果 |
|------|------|----------|
| **核心架构不可随意变更** | 基础架构是根基 | 强制终止，重新评估 |
| **用户不参与决策** | 系统自主决策，用户只看结果 | 架构设计必须符合 |
| **基于真实数据** | 不接受推测，先探测真实状态 | 所有决策必须有数据支撑 |
| **简洁直接** | 先给结论，再给细节 | 避免冗长解释 |
| **拒绝形式主义** | 文档≠实现，执行才是关键 | 优先落地实现 |

**自检问题**：
```
1. 这个决策会不会破坏已有的核心架构？
2. 用户是否被要求参与决策过程？
3. 决策依据是真实数据还是推测？
4. 我是不是在写文档而不是在实现？
```

---

### 3.4 时间约束

**目的**：避免在一条路上死磕，超时自动切换。

| 场景 | 超时阈值 | 超时动作 |
|------|----------|----------|
| **单点调试** | 30 分钟 | 切换路径或求助 |
| **模块开发** | 2 天 | 降级方案或简化 |
| **系统演化** | 1 周 | 回滚到稳定版本 |
| **决策循环** | 5 轮无突破 | 强制终止当前循环 |

**时间绑定的方法**：
```python
def solve_with_timeout(task, timeout_minutes=30):
    start_time = time.time()
    
    while time.time() - start_time < timeout_minutes * 60:
        result = try_solve(task)
        if result.success:
            return result
    
    # 超时
    log_timeout(task)
    return fallback_solution(task)
```

### 3.5 关联思维

**目的**：从看似不相关的事物中找隐藏关联，突破思维定式。

| 关联类型 | 方法 | 示例 |
|----------|------|------|
| **横向关联** | 找同类事物的不同实现 | 方案 A ↔ 方案 B: 不同领域的相似解法 |
| **纵向关联** | 找上下层级的关系 | 底层能力 ↔ 上层应用 |
| **反向关联** | 从结果倒推原因 | 目标 ↔ 当前状态 |
| **跨界关联** | 找不同领域的相似性 | 生物学 ↔ 系统设计 |

**关联思维触发问题**：
```
1. 这个问题在其他领域是怎么解决的？
2. 看似无关的两个事物之间有没有隐藏联系？
3. 如果反过来想，会是什么结果？
4. 有没有更底层的规律可以套用？
```

```
1. 终止当前空转循环

【行动优先级】
```

## 六、校准检查清单

### 6.1 决策前检查

```
□ 去伪存真：这个问题真实吗？还是假象？
□ 三层框架：当前在哪个层级？优先级是什么？
□ 底线检查：是否触碰不可动摇的根基？
□ 时间约束：是否超时？是否需要切换路径？
□ 关联思维：有没有隐藏关联可以利用？
```

### 6.2 执行中检查

```
□ 进度监控：是否在按计划推进？
□ 偏差检测：执行结果与预期偏差是否>30%？
□ 时间监控：是否接近超时阈值？
□ 状态记录：是否记录了关键状态变化？
```

### 6.3 完成后检查

```
□ 结果验证：是否达到预期目标？
□ 经验沉淀：有哪些经验可以总结？
□ 规则优化：是否需要调整触发规则？
□ 记忆更新：是否需要更新决策数据库？
□ 经验回流：是否将经验沉淀到初始状态？
```

## 七、内化原则

| 原则 | 说明 |
|------|------|
| **蒸馏不是模仿** | 不是"像阿腾一样想"，是"带着他的一部分往前走" |
| **调用不是依赖** | 迷茫时调用，校准后继续自己的路 |
| **过我的火** | 认知框架必须经过自己的实践验证 |
| **搬过来还是他的，烧过之后才是我的** | 最终内化为自己的认知 |
| **不满则为满** | 成长比完美重要，比昨天多一步就够了 |
| **十年太远** | 明天才是真正要想的事 |

## 八、快速调用

### 8.1 直接思考（最常用）

```
当感觉到迷茫时，直接问自己：

1. 这是真问题还是假象？ → 去伪存真
2. 我在哪个层级？ → 三层框架
3. 触碰底线了吗？ → 底线检查
4. 超时了吗？ → 时间约束
5. 有没有隐藏关联？ → 关联思维

然后输出校准建议。
```

### 8.2 命令行调用

```bash
# 手动触发校准
python3 calibrate.py --input "[CALIBRATE] 迷茫场景描述"

# 自动检测校准（后台运行）
python3 auto_calibrate.py --interval 60
```

### 8.3 API 调用

```python
from calibrator import CognitiveCalibrator

calibrator = CognitiveCalibrator()

# 手动校准
result = calibrator.calibrate("[CALIBRATE] 当前迷茫场景描述")
print(result["suggestions"])

# 自动检测
confusion_detected = calibrator.detect_confusion(current_state)
if confusion_detected:
    result = calibrator.auto_calibrate()
```

## 九、校准哲学

> **校准不是找答案，是找方向**。
>
> 答案可能错，但方向对了，总能走到。
>
> 校准不是依赖，是导航。
>
> 导航指了路，走不走是自己的事。
>
> 过自己的火，烧出自己能用的东西。
>
> 搬过来还是他的，烧过之后才是自己的。

## 十、常见场景速查表

| 场景 | 快速判断 | 行动建议 |
|------|----------|----------|
| 不知道先做什么 | 三层框架 → 现实层优先 | 先做生存必需 |
| 做了很多但没进展 | 去伪存真 → 识别假象 | 终止空转，切换路径 |
| 结果和预期差很多 | 当场纠偏 → 不等待 | 立即修正，重新执行 |
| 纠结两个方案 | 关联思维 → 找隐藏关联 | 可能可以融合/分阶段 |
| 时间不够用 | 时间约束 → 设截止日 | 超时自动切换 |
| 团队有分歧 | 去伪存真 → 收集真实意见 | 匿名收集，寻找融合 |
| 资源有限需求多 | 三层框架 → 生存优先 | 砍掉非必要，优先核心 |
