## 二、实事求是

### 2.1 核心思想

**"实事"就是客观存在着的一切事物，"是"就是客观事物的内部联系，即规律性，"求"就是我们去研究。**

```
┌─────────────────────────────────────────────────────────┐
│                  实事求是工作流程                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 调查研究                                              │
│     ↓ (了解实际情况)                                     │
│  2. 分析矛盾                                              │
│     ↓ (找出主要矛盾)                                     │
│  3. 制定策略                                              │
│     ↓ (理论联系实际)                                     │
│  4. 实践检验                                              │
│     ↓ (实践是检验真理的唯一标准)                         │
│  5. 总结经验                                              │
│     ↓ (在实践中发展理论)                                 │
│  6. 指导实践                                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **不唯书** | 不盲目相信书本 | 教条主义 |
| **不唯上** | 不盲目服从上级 | 官僚主义 |
| **只唯实** | 只相信客观实际 | 实事求是 |

### 2.3 在 StarCore 中的应用

```
│              StarCore 实事求是                            │
```

## 五、矛盾论

### 5.1 核心思想

**"事物的矛盾法则，即对立统一的法则，是唯物辩证法的最根本的法则。"**

```
┌─────────────────────────────────────────────────────────┐
│                  矛盾分析法                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 识别矛盾                                              │
│     ↓ (找出系统中的矛盾)                                 │
│  2. 区分主次                                              │
│     ↓ (主要矛盾 vs 次要矛盾)                             │
│  3. 分析转化                                              │
│     ↓ (矛盾双方相互转化)                                 │
│  4. 抓住重点                                              │
│     ↓ (集中力量解决主要矛盾)                             │
│  5. 统筹兼顾                                              │
│     ↓ (兼顾次要矛盾)                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.2 矛盾类型

| 类型 | 说明 | 例证 |
|------|------|------|
| **主要矛盾** | 决定事物发展方向 | Tweak 注入是核心问题 |
| **次要矛盾** | 影响但不决定 | SSH 隧道稳定性 |
| **矛盾的主要方面** | 决定矛盾性质 | Tweak 文件存在是主要方面 |
| **矛盾的次要方面** | 影响矛盾表现 | 注入机制缺失是次要方面 |

### 5.3 在 StarCore 中的应用

```
┌─────────────────────────────────────────────────────────┐
│              StarCore 矛盾分析                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  当前矛盾分析：                                          │
│                                                         │
│  主要矛盾：                                              │
│  • Tweak 注入机制不完善                                  │
│  • 解决：完善 ElleKit 配置，验证注入                     │
│                                                         │
│  次要矛盾：                                              │
│  • SSH 隧道偶尔断开                                      │
│  • 解决：增加心跳检测                                    │
│                                                         │
│  矛盾的主要方面：                                        │
│  • Tweak 文件已存在（积极方面）                          │
│  • 注入机制缺失（消极方面）                              │
│                                                         │
│  解决策略：                                              │
│  • 集中力量解决主要矛盾（Tweak 注入）                    │
│  • 兼顾次要矛盾（SSH 稳定性）                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 六、实践论

### 6.1 核心思想

**"通过实践而发现真理，又通过实践而证实真理和发展真理。"**

```
┌─────────────────────────────────────────────────────────┐
│                  实践 - 认识循环                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  实践 ──→ 感性认识 ──→ 理性认识 ──→ 再实践              │
│    ↑                                      ↓             │
│    └──────── 检验真理 ←─────────────────┘               │
│                                                         │
│  循环层次：                                              │
│  1. 第一次飞跃：实践 → 认识（感性→理性）                 │
│  2. 第二次飞跃：认识 → 实践（指导行动）                  │
│  3. 循环往复：实践、认识、再实践、再认识                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 6.2 核心原则

| 原则 | 说明 | 应用 |
|------|------|------|
| **实践第一** | 实践是认识的来源 | 先实测，再总结 |
| **实践检验** | 实践是检验真理的标准 | 功能上线验证 |
| **认识发展** | 认识在实践中发展 | 持续迭代优化 |

### 6.3 在 StarCore 中的应用

```
┌─────────────────────────────────────────────────────────┐
│              StarCore 实践论                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第一次飞跃（实践 → 认识）：                             │
│  1. 实践：尝试 Tweak 注入                                │
│  2. 感性认识：Tweak 文件存在但未加载                     │
│  3. 理性认识：ElleKit 配置缺失，SpringBoard 未链接       │
│                                                         │
│  第二次飞跃（认识 → 实践）：                             │
│  1. 制定方案：检查 ElleKit 配置，重启 SpringBoard        │
│  2. 执行方案：通过 SSH 验证                              │
│  3. 检验结果：Tweak 是否成功加载                         │
│                                                         │
│  循环往复：                                              │
│  • 若成功 → 总结经验，完善流程                           │
│  • 若失败 → 重新分析，调整方案                           │
│                                                         │
```
