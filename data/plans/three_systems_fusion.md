## 三、融合机制详解

### 3.1 输入路由机制

```python
def route_input(message: str) -> Dict:
    """
    输入路由：根据前缀决定进入哪个系统
    
    前缀规范：
    - 无前缀 → user（真实用户输入）
    - [SYSTEM] → system（系统自循环反馈）
    - [MCP]/[DEVICE] → external（外部工具返回）
    - [GUA] → hexagram（六十四卦状态查询/命令）
    - [HERMES] → hermes（直接工具调用）
    - [ATENG] → ateng（阿腾认知核心校准请求）
    """
    if message.startswith("[HERMES]"):
        # 直接调用 Hermes 工具
        return {"target": "hermes", "action": "tool_call", "content": message[8:]}
    
    elif message.startswith("[ATENG]"):
        # 阿腾认知核心校准请求 — 迷茫时调用
        return {"target": "ateng", "action": "calibrate", "content": message[7:]}
    
    elif message.startswith("[SYSTEM]"):
        # 两仪循环内部反馈
        return {"target": "two_yin_yang", "action": "feedback", "content": message[8:]}
    
    elif message.startswith("[MCP]") or message.startswith("[DEVICE]"):
        # 外部工具返回
        return {"target": "two_yin_yang", "action": "external_feedback", "content": message[5:]}
    
    elif message.startswith("[GUA]"):
        # 六十四卦状态查询/命令
        return {"target": "hexagram", "action": "command", "content": message[5:]}
    
    else:
        # 无前缀 = 真实用户输入 → 进入两仪循环阳端
        return {"target": "two_yin_yang", "action": "user_input", "content": message}
```

### 3.2 循环计数器融合

```python
class FusionCycleCounter:
    """
    融合循环计数器：统一管理三系统的循环状态
    """
    def __init__(self, db_path: str):
        self.db_path = db_path
        self.max_cycles = 10
        
        # 三层计数器
        self.user_input_count = 0      # 用户输入计数
        self.system_cycle_count = 0    # 系统自循环计数
        self.hexagram_evolution_count = 0  # 卦象演化计数
    
    def process_input(self, source: str) -> Dict:
        """处理输入，返回控制指令"""
        if source == "user":
            # 用户输入：重置所有计数器
            self.user_input_count = 0
            self.system_cycle_count = 0
            return {
                "action": "reset_all",
                "message": "用户输入，重置所有循环计数器"
            }
        
        elif source == "system":
            # 系统反馈：递增两仪循环计数
            self.system_cycle_count += 1
            if self.system_cycle_count >= self.max_cycles:
                return {
                    "action": "terminate",
                    "reason": f"两仪循环达到上限 ({self.max_cycles})",
                    "fallback": "六十四卦接管"
                }
            return {
                "action": "continue",
                "counter": self.system_cycle_count
            }
        
        elif source == "hexagram":
            # 卦象演化：独立计数
            self.hexagram_evolution_count += 1
            return {
                "action": "evolve",
                "counter": self.hexagram_evolution_count
            }
```

### 3.3 决策 - 执行 - 反馈闭环

```
用户输入（无前缀）
    ↓
两仪循环阳端：输出 5 个方向
    ↓
两仪循环阴端：8 维评估，选择最优
    ↓
Hermes 执行：调用工具执行决策
    ↓
执行结果 → [SYSTEM] 前缀回传
    ↓
两仪循环接收反馈 → 计数器递增
    ↓
如果计数器 < 10：继续下一轮
如果计数器 >= 10：切换到六十四卦演化模式
    ↓
六十四卦：根据熵/能量触发卦象跃迁
    ↓
卦象变化 → [GUA] 前缀通知两仪循环
    ↓
两仪循环根据新卦象调整方向
    ↓
循环继续...
```

## 四、关键融合点

### 4.1 触发条件融合

| 触发源 | 六十四卦 | 两仪循环 | Hermes |
|--------|---------|---------|--------|
| 用户输入 | 不直接响应 | ✅ 启动阳端 | ✅ 执行工具 |
| 高熵 (>0.6) | ✅ 触发变爻 | ⚠️ 建议深度评估 | - |
| 低能量 (<30%) | ✅ 触发恢复卦 | ✅ 建议休息 | - |
| 循环达上限 | ✅ 接管演化 | ❌ 强制终止 | - |
| 工具执行完成 | - | ✅ 接收反馈 | ✅ 返回结果 |

### 4.2 前缀规范统一

| 前缀 | 来源 | 目标系统 | 计数器行为 |
|------|------|---------|-----------|
| (无前缀) | 用户 | 两仪循环 | 重置 |
| `[SYSTEM]` | 两仪内部 | 两仪循环 | 递增 |
| `[MCP]` | 外部工具 | 两仪循环 | 保持 |
| `[GUA]` | 六十四卦 | 全系统 | 通知 |
| `[HERMES]` | 用户/外部 | Hermes | 直接执行 |

### 4.3 故障转移机制

```
主路径：用户 → 两仪循环 → Hermes 执行 → 反馈
    ↓ (两仪循环达上限)
备用路径：六十四卦演化 → 卦象跃迁 → 通知两仪循环 → 继续
    ↓ (六十四卦无触发条件)
静默模式：保持当前状态，等待用户输入
```
