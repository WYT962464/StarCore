# 两仪循环引擎 v5.1 - 真实回传机制说明

## 一、核心原则

**真实回传 = 执行结果作为反馈**

| 要素 | 说明 |
|------|------|
| **来源** | 必须来自真实执行结果（iOS MCP、iOS Controller） |
| **方式** | 通过系统 API 获取，非人为模拟 |
| **目的** | 反馈驱动下一轮决策，形成真实闭环 |
| **限制** | 在循环步数限制内完成，不可绕过 |

## 三、反馈数据来源

| 数据源 | API | 用途 |
|--------|-----|------|
| 屏幕截图 | `POST /screenshot` | 视觉验证执行效果 |
| 前台应用 | `POST /frontmost` | 验证应用切换 |
| 设备状态 | `GET /state` | 系统健康检查 |
| 执行日志 | `GET /health` | 服务状态追踪 |

---

## 四、循环步数限制

| 参数 | 值 | 说明 |
|------|-----|------|
| `max_cycles` | 10 | 单轮决策最大循环次数 |
| `cycle_timeout` | 300s | 单循环最大执行时间 |
| `feedback_threshold` | 0.7 | 反馈质量阈值 |

**终止条件**:
1. 达到最大循环次数
2. 反馈质量低于阈值
3. 决策置信度过低
4. 连续相同决策（可能陷入局部最优）

---

## 五、关键区别

| 错误做法 | 正确做法 |
|---------|---------|
| 模拟用户在对话窗口发送消息 | **执行结果作为真实反馈** |
| 绕过循环步数限制 | 在限制内完成闭环 |
| 人为触发下一轮 | **反馈自主驱动下一轮** |
| 虚假反馈数据 | **真实 API 响应数据** |

---

## 六、实施要点

1. **反馈必须真实**: 来自 iOS MCP/iOS Controller API 的实际响应
2. **不可绕过限制**: 循环步数计数器必须递增，达到上限自动终止
3. **反馈进入上下文**: 反馈数据必须影响下一轮阳端方向生成
4. **可审计追溯**: 所有反馈来源、时间、内容必须记录到数据库
